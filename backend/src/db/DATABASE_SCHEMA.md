# Database Schema Documentation

## Overview
The Smart Supply Chain database uses PostgreSQL with PostGIS extension for geographic queries. The schema is designed to support a logistics digital twin with real-time shipment tracking, disruption detection, AI-powered predictions, and route optimization.

## Tables

### 1. shipments
Core table storing all shipment information and current state.

| Column | Type | Description |
|--------|------|-------------|
| id | SERIAL | Primary key |
| tracking_number | VARCHAR | Unique shipment identifier |
| origin | VARCHAR | Origin address/location name |
| destination | VARCHAR | Destination address/location name |
| status | VARCHAR | Current status: PENDING, IN_TRANSIT, AT_HUB, DELIVERED, DELAYED |
| priority | VARCHAR | Shipment priority: LOW, MEDIUM, HIGH, URGENT |
| latitude | FLOAT | Current latitude position |
| longitude | FLOAT | Current longitude position |
| current_location | JSONB | Full location data: {lat, lng, timestamp} |
| carrier | VARCHAR | Carrier/transport company name |
| carrier_id | INTEGER | Foreign key to carriers table (future) |
| origin_warehouse_id | INTEGER | FK to warehouses table |
| destination_warehouse_id | INTEGER | FK to warehouses table |
| distance_km | FLOAT | Total route distance |
| estimated_time_hours | FLOAT | Original estimated delivery time |
| current_speed_kmh | FLOAT | Current speed of shipment |
| shipment_type | VARCHAR | STANDARD, EXPRESS, FRAGILE, HAZMAT |
| weight_kg | FLOAT | Shipment weight |
| cargo_details | JSONB | {items: [], value: 0, temperature_control: false} |
| estimated_delivery | TIMESTAMP | Estimated delivery datetime |
| actual_delivery | TIMESTAMP | Actual delivery datetime |
| created_at | TIMESTAMP | Record creation time |
| updated_at | TIMESTAMP | Last update time |

**Indexes:**
- status
- created_at
- carrier_id
- origin_warehouse_id, destination_warehouse_id
- (status, priority) - composite


## PostGIS Geographic Queries

The schema uses PostGIS for efficient geographic queries:

1. **Find shipments near a location:**
```sql
SELECT * FROM shipments
WHERE ST_DWithin(
  ST_SetSRID(ST_MakePoint(longitude, latitude), 4326)::geography,
  ST_SetSRID(ST_MakePoint(-74.0060, 40.7128), 4326)::geography,
  50000  -- 50km radius
);
```

2. **Find warehouses within disruption area:**
```sql
SELECT w.* FROM warehouses w
JOIN disruption_areas da ON
  ST_DWithin(w.location, da.center_location, da.affected_radius_km * 1000);
```

3. **Calculate distance between warehouses:**
```sql
SELECT
  w1.name, w2.name,
  ST_Distance(w1.location, w2.location) / 1000 as distance_km
FROM warehouses w1, warehouses w2
WHERE w1.id < w2.id;
```

## Environment Setup

Required environment variables in `.env`:
```
DATABASE_URL=postgresql://user:password@localhost:5432/supply_chain_db
```

For PostGIS support, ensure the extension is enabled:
```sql
CREATE EXTENSION IF NOT EXISTS postgis;
```

## Performance Considerations

1. **Indexes**: All frequently queried columns have indexes
2. **Connection Pooling**: Configured for 2-10 concurrent connections
3. **Geographic Queries**: Use PostGIS geography type for accurate distance calculations
4. **JSONB**: Used for flexible schema parts (metadata, configurations)
5. **Timestamps**: Automatic `created_at` and `updated_at` tracking
