## Overview

ChainMind uses **9 serialized joblib model files** across 4 prediction domains:

1. **Delay Prediction** — Will a shipment be delayed? By how long?
2. **Anomaly Detection** — Are current metrics abnormal?
3. **Route Scoring** — Which route candidate is optimal?
4. **Disruption Simulation** — What impact will a disruption cause?

All models are trained on **30,000 synthetic records** generated by `training/generate_data.py`, with deterministic target variables computed from non-linear feature interactions to ensure realistic patterns without data leakage.

The backend includes **heuristic fallback** implementations that activate when the AI service is unreachable, ensuring predictions are always available.

## Synthetic Data Generation

**File:** `training/generate_data.py` (185 lines)
**Output:** `training/data/supply_chain_data.csv` — 30,000 records

### Generation Strategy

The data generator creates realistic supply chain records with **deterministic target variables** — targets are computed directly from features using non-linear mathematical formulas, not random sampling. This ensures the trained ML models learn actual causal patterns.

### Generated Features

| Feature              | Type        | Range                                                       | Distribution   |
| -------------------- | ----------- | ----------------------------------------------------------- | -------------- |
| `origin`             | Categorical | 10 cities                                                   | Uniform        |
| `destination`        | Categorical | 10 cities                                                   | Uniform        |
| `shipment_type`      | Categorical | `standard`, `express`, `fragile`, `hazardous`, `perishable` | Weighted       |
| `carrier`            | Categorical | 4 carriers                                                  | Uniform        |
| `distance_km`        | Float       | 100–5000                                                    | Random uniform |
| `weather_risk`       | Float       | 0.0–1.0                                                     | Random uniform |
| `traffic_risk`       | Float       | 0.0–1.0                                                     | Random uniform |
| `shipment_weight_kg` | Float       | 10–10,000                                                   | Random uniform |

### Cities

```
Mumbai, Delhi, Chennai, Kolkata, Bangalore,
Hyderabad, Ahmedabad, Pune, Jaipur, Lucknow
```

### Deterministic Target Formulas

#### `is_delayed` (Binary)

```python
delay_score = (
    weather_risk * 0.35 +
    traffic_risk * 0.25 +
    (distance_km / 5000) * 0.15 +
    type_risk_map[shipment_type] * 0.15 +
    (weight_kg / 10000) * 0.10
)
# Add non-linear interaction term
delay_score += weather_risk * traffic_risk * 0.2
is_delayed = delay_score > 0.45  # threshold
```

**Type Risk Map:**
| Type | Risk Weight |
|------|------------|
| perishable | 0.8 |
| hazardous | 0.7 |
| fragile | 0.5 |
| express | 0.3 |
| standard | 0.2 |

#### `delay_minutes` (Regression)

```python
delay_minutes = max(0, (delay_score - 0.3) * 500 + weather_risk * 120 + traffic_risk * 80)
# Add noise: ±20%
```

#### `delay_days` (Regression)

```python
delay_days = delay_minutes / (24 * 60)  # Convert minutes to fractional days
```

## Training Pipeline

**File:** `training/train_models.py` (281 lines)

### Execution

```bash
cd ai-service
python training/train_models.py
```

### Pipeline Steps

1. **Load data** from `training/data/supply_chain_data.csv`
2. **Encode categorical features** using LabelEncoder (fit + save)
3. **Train/test split** — 80/20 with `random_state=42`
4. **Train Delay Classifier** — RandomForestClassifier with 5-fold CV
5. **Train Delay Regressors** — GradientBoostingRegressor (minutes + days)
6. **Train Anomaly Detector** — IsolationForest on scaled numerical features
7. **Train Route Score Regressor** — GradientBoostingRegressor on route features
8. **Train Disruption Simulators** — GradientBoostingRegressor (shipments + delay)
9. **Save all models** to `app/models/` as joblib files
10. **Print evaluation metrics** for each model

## Model 5: Anomaly Detector

**Purpose:** Detect abnormal metric patterns that may indicate system issues, fraud, or unusual behavior.

| Property      | Value                                              |
| ------------- | -------------------------------------------------- |
| Algorithm     | `IsolationForest`                                  |
| contamination | 0.1 (10% expected anomalies)                       |
| n_estimators  | 100                                                |
| random_state  | 42                                                 |
| Scaler        | `StandardScaler` (fitted on training data)         |
| Files         | `anomaly_detector.joblib`, `anomaly_scaler.joblib` |

### Input (8 metrics)

| Metric                | Description                   |
| --------------------- | ----------------------------- |
| `delivery_time_hours` | Actual delivery duration      |
| `distance_km`         | Route distance                |
| `weather_risk`        | Current weather risk score    |
| `traffic_risk`        | Current traffic risk score    |
| `cost_usd`            | Shipment cost                 |
| `weight_kg`           | Payload weight                |
| `delay_hours`         | Current delay                 |
| `route_changes`       | Number of route modifications |

### Inference Logic

1. Scale input metrics using `anomaly_scaler`
2. Call `anomaly_detector.decision_function()` → raw anomaly score
3. Normalize: `anomaly_score = 1 - (raw_score - min) / (max - min)`
4. Compare against threshold (default: 0.7)
5. Rank top indicators by deviation from mean

### Performance

| Metric              | Value |
| ------------------- | ----- |
| Accuracy            | 91%   |
| Precision (anomaly) | ~0.88 |
| Recall (anomaly)    | ~0.90 |

### Output

```json
{
  "anomaly_score": 0.82,
  "is_anomaly": true,
  "threshold": 0.7,
  "top_indicators": [
    { "metric": "delivery_time_hours", "value": 142.5, "deviation": 3.2 },
    { "metric": "delay_hours", "value": 48.0, "deviation": 2.8 }
  ]
}
```

## Inference Services

### Prediction Service (`app/services/prediction_service.py` — 155 lines)

Handles delay prediction across all 4 delay models:

```
Input → Label Encode → Classifier (is_delayed, probability)
                     → Regressor minutes (predicted_delay_min)
                     → Regressor days (predicted_delay_days)
                     → Risk Level computation
                     → Top factor extraction
```

**Risk Level Mapping:**
| Probability | Level |
|-------------|-------|
| < 0.3 | low |
| 0.3 – 0.6 | medium |
| 0.6 – 0.8 | high |
| ≥ 0.8 | critical |

**Top Factors:** Extracted from feature importances of the classifier, sorted by weight.

### Disruption Detector (`app/services/disruption_detector.py` — 125 lines)

Handles anomaly detection using IsolationForest + StandardScaler:

```
Input metrics → Scale → IsolationForest decision_function → Normalize → Rank indicators
```

### Route Optimizer (`app/services/route_optimizer.py` — 170 lines)

Scores and recommends routes:

```
Candidates[] → Normalize features → Score (0-1) → Sort → Top 3 recommendation
```

### Simulation Service (`app/services/simulation_service.py` — 96 lines)

Estimates disruption impact:

```
Scenario params → Encode type → Predict affected shipments → Predict average delay
```

## Backend Heuristic Fallbacks

When the Python AI service is unreachable, the backend (`services/aiService.ts`) uses heuristic models:

### Delay Prediction Heuristic

```
base_probability = 0.05
  + weather_risk × 0.25
  + traffic_risk × 0.25
  + (distance_km / 10000) × 0.15
  + disruption_count × 0.05
  + (congestion_score × 0.1)

delay_minutes = probability × planned_eta_minutes
  + active_disruptions × 22
  + weather_risk × 30
```

### Route Scoring Heuristic

```
For each candidate:
  penalties = {
    distance: (dist / max_dist) × weight_distance,
    duration: (dur / max_dur) × weight_time,
    cost: (cost / max_cost) × weight_cost,
    risk: risk_score × weight_risk,
    stops: (stops / 20) × weight_stops
  }
  score = 1 - sum(penalties)
```

### Anomaly Detection Heuristic

```
normalized_metrics = average of all input metric values (0-1 scaled)
anomaly_score = mean(normalized_metrics)
is_anomaly = anomaly_score > threshold
```

## Anti-Overfitting Measures

The training pipeline includes multiple anti-overfitting safeguards:

| Technique                   | Applied To       | Details                                          |
| --------------------------- | ---------------- | ------------------------------------------------ |
| **5-fold Cross Validation** | Delay Classifier | Validates generalization before saving           |
| **max_depth limits**        | All tree models  | Depth 6–12 prevents deep memorization            |
| **min_samples_split**       | All tree models  | ≥ 10 samples required to split                   |
| **min_samples_leaf**        | All tree models  | ≥ 5 samples in each leaf                         |
| **class_weight='balanced'** | Delay Classifier | Handles class imbalance                          |
| **Deterministic targets**   | Data generation  | Non-random target formulas prevent noise-fitting |
| **80/20 train/test split**  | All models       | Holdout validation on unseen data                |
| **fixed random_state=42**   | All models       | Reproducible splits and training                 |
