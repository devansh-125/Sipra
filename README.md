## Overview

**ChainMind** is a full-stack supply chain management platform that combines classical ML prediction models with Generative AI (Google Gemini via OpenRouter) to provide:

- **Predictive analytics** — delay probability, ETA regression, anomaly detection
- **Route intelligence** — scoring, comparison, and AI-driven rerouting
- **Disruption simulation** — synthetic scenario generation with cascading impact analysis
- **AI Chat (ChainMind)** — natural language interface with 10 function-calling tools that query live data
- **Autonomous Agent** — Observe → Reason → Act loop that detects issues and executes corrective actions
- **Executive Insights** — fleet, shipment, disruption, and daily briefing narratives generated by LLM

The platform tracks shipments across a graph-based transportation network (hubs, edges, routes) with real-time WebSocket updates and an interactive Leaflet map dashboard.

## Getting Started

### Prerequisites

- **Node.js** ≥ 18
- **Python** ≥ 3.10
- **PostgreSQL** (or a Neon serverless account)
- **OpenRouter API Key** (for GenAI features)

### 1. Clone the Repository

```bash
git clone https://github.com/your-org/supply-chain.git
cd supply-chain
```

### 2. Set Up the Database

Create a Neon PostgreSQL database (or local Postgres) and note the connection string.

### 3. Configure Environment Variables

**Backend** (`backend/.env`):

```env
DATABASE_URL=postgresql://user:password@host/dbname?sslmode=require
AI_SERVICE_URL=http://localhost:8000
PORT=3001
DEMO_API_ENABLED=true
```

**AI Service** (`ai-service/.env`):

```env
OPENROUTER_API_KEY=sk-or-v1-your-key-here
BACKEND_URL=http://localhost:3001
LLM_MODEL=google/gemini-2.0-flash-001
```

**Frontend** (`frontend/.env`):

```env
VITE_API_BASE_URL=http://localhost:3001
```

### 4. Install Dependencies & Run Migrations

```bash
# Backend
cd backend
npm install
npm run migrate

# AI Service
cd ../ai-service
pip install -r requirements.txt

# Frontend
cd ../frontend
npm install
```

### 5. Start All Services

Open three terminals:

```bash
# Terminal 1 — Backend
cd backend && npm run dev

# Terminal 2 — AI Service
cd ai-service && uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

# Terminal 3 — Frontend
cd frontend && npm run dev
```

### 6. Seed Demo Data

Once all services are running, use the Demo Controls in the dashboard or call:

```bash
curl -X POST http://localhost:3001/api/demo/seed
```

## Generative AI

Powered by **Google Gemini 2.0 Flash** via **OpenRouter**. See [GENERATIVE_AI_AND_AGENT.md](GENERATIVE_AI_AND_AGENT.md) for full details.

### Chat (ChainMind)

Natural language interface with 10 function-calling tools that query backend APIs for shipments, routes, disruptions, alerts, network nodes, and bottlenecks. Chains up to 10 tool rounds for complex queries.

### Autonomous Agent

Observe → Reason → Act → Report cycle:

1. **Observe** — Gathers shipments, disruptions, alerts, bottleneck data
2. **Reason** — LLM analyzes patterns and decides actions
3. **Act** — Executes reroutes, creates alerts, resolves disruptions
4. **Report** — Returns structured summary of observations and actions

### Narrative Insights

4 report types generated by LLM with structured JSON schemas:

- Fleet Overview — system-wide health and recommendations
- Shipment Deep-Dive — single shipment risk analysis
- Disruption Analysis — impact and mitigation strategies
- Executive Briefing — daily 6-section report

## Real-Time Events

Socket.IO events broadcast from the backend:

| Event                 | Payload                            | Trigger                   |
| --------------------- | ---------------------------------- | ------------------------- |
| `shipment:updated`    | `{ shipmentId, status }`           | Status change, reroute    |
| `shipment:delayed`    | `{ shipmentId }`                   | Status set to 'delayed'   |
| `shipment:delivered`  | `{ shipmentId }`                   | Status set to 'delivered' |
| `shipment:rerouted`   | `{ shipmentId, routePlanId }`      | New route plan activated  |
| `disruption:new`      | `{ disruptionId, type, severity }` | Disruption created        |
| `disruption:resolved` | `{ disruptionId }`                 | Disruption resolved       |
| `alert:new`           | `{ alertId, severity, title }`     | Alert created             |
| `dashboard:refresh`   | `{ reason }`                       | Any data mutation         |

## Troubleshooting

| Issue                           | Cause                             | Fix                                                                                  |
| ------------------------------- | --------------------------------- | ------------------------------------------------------------------------------------ |
| Charts show "Failed to fetch"   | DB connection pool exhausted      | Ensure `knexfile.ts` has `pool: { min: 0, max: 5 }`, `idleTimeoutMillis: 10000`      |
| AI Chat returns shallow answers | Single-tool usage                 | System prompt includes multi-tool chaining strategies; verify `MAX_TOOL_ROUNDS ≥ 10` |
| Agent reroute fails             | No active route plan for shipment | `rerouteShipment()` auto-generates initial route fallback                            |
| `geometry_json` column error    | JSONB needs string input          | `JSON.stringify()` applied in `routeService.ts`                                      |
| WSL DNS resolution fails        | Default DNS not working           | Set `/etc/resolv.conf` nameserver to `8.8.8.8`                                       |
| OpenAI SDK timeout              | OpenRouter latency via WSL        | Timeout set to 120 seconds in `gemini_service.py`                                    |
| `.env` not loading              | CRLF line endings in WSL          | Convert to LF: `sed -i 's/\r$//' .env`                                               |
